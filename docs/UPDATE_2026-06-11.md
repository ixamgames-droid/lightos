## 1. EFX: Custom Paths (eigene Bewegungen)

**Bedienung:** EFX-Bereich → Zeile **„Custom Path"** unter dem Algorithmus:
- **„+ Aufzeichnen…"** öffnet das Popout: Tippen/Klicken setzt Punkte,
  Punkte ziehen = verschieben, Liste rechts = auswählen/▲▼ umsortieren/
  löschen; **Linear oder Spline (weiche Kurve)**; „Pfad schließen";
  animierte Vorschau; Name + Speichern.
- Gespeicherte Pfade stehen in der **Auswahlliste** — Pfad wählen schaltet
  die EFX auf „Custom Path". **„Bearbeiten…"** ändert den Pfad (wirkt live
  auf laufende EFX), 🗑 löscht ihn aus der Bibliothek (laufende Shows
  behalten ihre eingebettete Kopie).
- **Loop** ist eine EFX-Eigenschaft im Hauptfenster („Endlos wiederholen",
  Gruppe Tempo & Richtung): aus = One-Shot, die Bewegung läuft einmal und
  hält am Endpunkt (bounce = einmal hin und zurück). Neustart über die
  Aktion „Neustart" oder Stop/Start.
- Größe/Zentrum/Rotation/Fan/Spiegeln wirken auf Custom Paths wie auf die
  eingebauten Formen; das Sampling ist **bogenlängen-parametrisiert**
  (konstante Geschwindigkeit, egal wie die Punkte verteilt sind).

**Persistenz:** Pfad-Bibliothek in der Show (`efx_paths`-Block, analog
Kurven); jede EFX speichert zusätzlich eine eingebettete Pfad-Kopie.

## 2. EFX über VC/MIDI

`EfxInstance` implementiert jetzt die Live-API (`list_params`/`set_param`/
`do_action`/`list_actions`) — dieselbe Mechanik wie Matrix/Chaser, **keine
Parallelstruktur**. Damit überall verfügbar:

- **Parameter** (Fader/Encoder, `EFFECT_PARAM`): Speed (Hz), Intensität,
  Größe (Pan+Tilt gemeinsam), Breite, Höhe, Zentrum Pan/Tilt, Rotation,
  Streuung (Fan), Richtung, Loop, Spiegeln, Beam öffnen, Form (Algorithmus),
  **Custom Path** (Auswahl per Fader/Select).
- **Aktionen** (Buttons, `EFFECT_ACTION`): Neustart/Reset, Loop an/aus,
  Richtung, Bounce, **Pfad +/−**, **Form +/−**, Spiegeln, Beam öffnen,
  **Auf Auswahl anwenden**, Tap-Tempo.
- Start/Stop wie gehabt über FunctionToggle/-Flash; Blackout/Stop All über
  die bestehenden VC-Aktionen.

**Einrichten:** VC-Widget an die EFX binden → Rechtsklick →
**„⚡ Live-Parameter…"** erzeugt automatisch Fader+Tasten (der Dialog zeigt
jetzt die zur Funktion passenden Aktionen); MIDI-Learn pro Widget wie immer.
Mappings liegen im VC-Layout der Show. Bestehende Chaser-/Matrix-Mappings
unverändert (Regression getestet).

## 3. Patchen: „Gruppe bearbeiten"

Patchen → Gruppenansicht → Button **„Bearbeiten…"**: Dialog mit
Mitgliederliste (geordnet) ↔ verfügbare Fixtures (Doppeltipp = hinzufügen),
▲/▼ = Reihenfolge (relevant für Fan/Chase), Name ändern (Duplikat-Rückfrage),
Speichern/Abbrechen. Ohne Umsortierung bleiben die Raster-Zellen erhalten
(2D-Layout für die Matrix); Umsortieren schreibt das Raster zeilenweise neu
(Hinweis im Dialog). Änderungen gehen über `GROUP_CHANGED` an Programmer,
Live View, EFX und Matrix; Persistenz in Show-DB + .lshow.

## 4. Live Viewer: Touch-Auswahl

War großteils vorhanden (Mehrfachauswahl-Modus, „＋ Gruppe aus Auswahl",
„Auswahl leeren", Listen-Sync). Neu:
- Im Mehrfachauswahl-Modus schaltet auch die **linke Fixture-Liste** auf
  Tipp-Toggle um (kein Strg/Shift nötig); aus = gewohntes Verhalten.
- **Größere Trefferflächen**: im Modus mindestens ~24 Bildschirm-Pixel
  Hit-Radius (zoom-unabhängig), und es gewinnt das **nächste** Fixture
  (nicht das erste in interner Reihenfolge) — wichtig bei dichten Rigs.
- Versehentliche Auswahl: Tipp im Modus toggelt nur (kein Drag), Rubber-Band
  bleibt additiv, Leeren über den Button.

## 5. Programmer: Ordner auf-/zuklappen

- **Gruppen-Liste:** 📁-Kopfzeilen sind antippbar (▾/▸) und klappen ihre
  Gruppen ein/aus; Zustand persistiert in `ui_prefs.json`.
- **Bibliothek (Snaps/Effekte):** eingeklappte Ordner überleben jetzt die
  automatischen Baum-Rebuilds (FUNCTION_CHANGED) und den Neustart
  (`library_collapsed_folders` in `ui_prefs.json`). Vorher klappte jeder
  Refresh alles wieder auf.

## 6. Controller-Datenbank

- **Datenmodell/Bibliothek:** `src/core/controllers/controller_library.py`,
  JSON-Profile in `data/controller_library/` (Builtins) +
  `%APPDATA%/LightOS/controller_library/` (Importe). Felder: Hersteller,
  Modell, Typ, Anschluss, Tasten/Fader/Encoder, Pad-Matrix, MIDI-Belegung
  (Notes/CCs/Kanäle), LED-Feedback, Banks, Besonderheiten, Quelle, Lizenz,
  Importdatum, VC-/Mapping-Vorlage. Gleiche id wird nie überschrieben.
- **Seed-Daten (8):** APC mini, APC mini mk2, KORG nanoKONTROL2, Behringer
  X-Touch Mini, Novation Launchpad Mini MK3, Enttec DMX USB Pro, generischer

  und gesammelt in `data/controller_library/README.md`.
- **UI:** MIDI-Konsole → **„Controller-Profile…"** — Liste + Details

  Input-Profil).

## 8. VC: Keyboard-Mapping

Rechtsklick auf VC-Button → **„⌨ Taste zuweisen…"** → Taste/Kombination
drücken → Übernehmen. Learn-Modus, Konfliktprüfung, Kritisch-Kennzeichnung
(Blackout/Stop), Persistenz im VC-Layout, Textfeld-/Dialog-Schutz,
Flash-fähig (Press/Release wie MIDI-Note). Details + Grenzen der
Zweittastatur-Erkennung: `docs/KEYBOARD_MAPPING.md`.

grün.
